2.4.- Transformación de los datos¶

Los rboles de decisin que proporciona scikit learn son de tipo CART, por lo que no estan admitidos los datos categricos ni nominales, slo numricos.

Es necesario mapear y/o codificar los valores de todos los atributos de la base de datos con nmeros. Primero se crean los diccionarios correspondientes a cada atributo, que incluye el valor actual junto a su respectivo cdigo , y luego se utiliza el mtodo map() para reemplazar todos los valores en cada atributo.

In [5]:
#Lista que se utilizar para far formato al rbol de decisin
cars_feature_names = ["buying", "maint", "doors", "persons", "lug_boot","safety"]

cod = {"low":1, "med":2, "high":3, "vhigh":4} #para buying y maint
safety_cod = {"low":1, "med":2, "high":3} #para safety
lug_boot_cod = {"small":1, "med":2, "big":3} #para lug_boot
class_cod = {"unacc":1, "acc":2, "good":3, "vgood":4} #para class
doors_cod = {"1":1, "2":2, "3":3, "4":4, "5more":5} #para doors
persons_cod = {"2":1, "4":2, "more":3} #para persons

#Mapeo de datos
carsCSV["buying"] = carsCSV["buying"].map(cod)
carsCSV["maint"] = carsCSV["maint"].map(cod)
carsCSV["safety"] = carsCSV["safety"].map(safety_cod)
carsCSV["lug_boot"] = carsCSV["lug_boot"].map(lug_boot_cod)
carsCSV["class"] = carsCSV["class"].map(class_cod)
carsCSV["doors"] = carsCSV["doors"].map(doors_cod)
carsCSV["persons"] = carsCSV["persons"].map(persons_cod)

2.5.- Minería de datos¶

2.5.1.- Partición de los datos¶

Al ser los rboles de decisiones algoritmos de aprendizaje supervisados, es necesario dividir el conjunto de datos para generar el modelo. Una parte ser para el entrenamiento o aprendizaje, que ser el 80% de todos los datos. La otra parte servir para ser evaluada por el rbol de prediccin, que por el momento no ser utilizada en sta prctica (esta parte corresponde al 20% restante).

La librera de scikit learn nos proporciona un mtodo que se encarga de particionar los datos, de forma balanceada y aleatoria, para evitar problemas y complicaciones que pudiesen surgir de un particionamiento manual.

Para los valores en x, eliminamos la columna class, ya que los valores en y sern los datos de ste atributo. Utilizando el mtodo train_test_split, que recibe como parmetros los valores x, los valores y, el tamao del conjunto de entrenamiento (0.8 = 80%), y una semilla aleatoria, se seleccionan aleatoriamente los mejores datos para entrenar al rbol de aprendizaje CART:

X_values = carsCSV.iloc[:,:6]

2.5.4.- Importar reglas de clasificación a sentencia SQL¶

Obtener una sentencia SQL a partir del modelo representado por las reglas de clasificacin del rbol de decisin CART generado es de gran ayuda cuando se tenga la base de datos cargada en un gestor de base de datos, como MySQL, y querramos hacer consultas para obtener registros que cumplan una serie de condiciones, logrando as clasificarlos en el proceso.

Cabe aclarar que sta herramienta an esta en desarrollo, por lo que no es posible personalizar la sentencia SQL, como por ejemplo asignar el nombre de los atributos, as que el servidor los asigna por defecto.

2.5.5.- Importar reglas de clasificación a PMLL¶

Para guardar el modelo del rbol CART generado representado por sus reglas de decisin en un archivo PMML, primero creamos un pipeline que servir como medio de comunicacin de escritura entre el modelo guardado en la ejecucin del cdigo Python, y el archivo en disco duro.

Finalmente, invocamos al mtodo sklearn2pmml(), que recibe como parmetros el pipeline y el nombre del archivo a escribir.

In [12]:
pipeline = PMMLPipeline([("cars_python", modelo)])

Cuando se requiera hacer una clasificacin con otros datos de la base de datos, este archivo PMML es til para no repetir todo el proceso para la creacin del modelo de clasificacin (rbol de decisin CART) descrito en este reporte desde el principio.

3.- Resultados encontrados¶

El atributo ms significativo de toda la base de datos para determinar el estado de un automvil es su seguridad, ya que si esta es baja o media, casi siempre el auto se clasificar en un estado inaceptable.

Por otro lado, el precio de venta y el costo de mantenimiento tambin juegan un papel importante. Si el precio de venta es bajo, pero el costo de mantenimiento es alto, el auto cae en clasificaciones de tipo aceptable o bueno, es decir, un estado intermedio que no es ni bueno ni malo. Si ambos atributos tienen costos muy altos, se clasificar como un auto no aceptable, a menos que tenga un maletero de tamao grande, un nmero de puertas mayor a 4, y/o una capacidad de personas de ms de 5; si la seguridad esta en un rango medio alto, se puede clasificar incluso en un buen estado, a pesar de los altos costos.

Los mejores autos son los que tengan un costo de mantenimiento y precio de venta medios, un maletero de gran tamao, capacidad para ms de 4 personas, y una seguridad alta.
